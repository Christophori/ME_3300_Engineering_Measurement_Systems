\section{Learning objective}
The objectives of this experiment are to provide the student with an opportunity to:

\begin{itemize}
    \item Gain additional experience with Matlab.
    \item Create a simple program for data acquisition using toolbox and National Instrument’s \href{https://www.ni.com/docs/en-US/bundle/mydaq-specs/page/specs.html}{NI-MyDAQ}.
    \item Become familiar with breadboards, digital power supply, wiring, digital multi-meters, and other
laboratory skills.
    \item Perform calibration of a MEMS accelerometer and create the calibration plot.
    \item Compare measured normal acceleration to acceleration calculated from angular velocity during fixed axis rotation.
\end{itemize}

\section{Introduction}

During rotation about a fixed axis, the normal acceleration of a point can be calculated from the angular velocity of the rotation motion according to the following formula:

\begin{equation}
a_{n} = -\dot{\theta}^2 r
\label{Eq:NormAcc}
\end{equation}

\section{Laboratory instruction}
In this experiment, you will build on your work from the previous week. In addition to measuring the pendulum angle with a calibrated potentiometer, you will also measure normal acceleration of the end of the pendulum using a MEMS accelerometer. The schematic is shown in figure \ref{fig:PendulumDiagram}. Start by setting up the NI MyDAQ to acquire data from the potentiometer and accelerometer. Next, you will calibrate the y-axis of the accelerometer. Finally, we will compare the measured normal acceleration of the pendulum end to the normal acceleration calculated from the derivative of the pendulum angle.

\section{Part 1: Prepare your MATLAB Program to Display and Record Data.}

\begin{enumerate}
    \item Launch MATLAB and prepare your code
    \begin{itemize}
        \item A main acquisition script \boldpurp{MATLAB\_Data\_Acquisition.m} is posted on canvas along with two supporting functions \boldpurp{plotdata2.m} \& \boldpurp{logData2.m}
        \item Toggle ``inputChannels" between single channel and multiple channel to initially calibrate the accelerometer and then collect the potentiometer data from ``ai0" and accelerometer data from ``ai1".
        \item You need to set the parameters for data acquisition in the main program. They are set near the top of the file.
    \end{itemize} 
    \item Set the value of Fs to set the sample rate (the number of samples to collect per second). 
    \begin{itemize}
        \item The MyDAQ can collect up to 200,000 samples per second, but this would produce a massive data file \& we don't need that much resolution to observe the pendulum swing. 
        \item You will set this value differently for each part of the lab. Note what settings are best for different tasks. 
        \item For the final data acquisition, set the sample rate to 1000 samples/sec. We will be calculating a discrete derivative of the pendulum angle, and a higher sample rate will reduce the noise from the process.
        \item [\textbf{Q}] Why does a higher sample rate reduce the noise in the calculated derivative? Why does it increase the size of data files?
    \end{itemize}
    \item Set T to control the duration of your acquisition. This tells the DAQ how much time in seconds to collect data.
    \item \boldblue{inputChannels} controls the collected input signals. An example of collecting 2 channels is provided.
    \item The two boolean variables \boldblue{enableLogging} \& \boldblue{enableLogging} control if statements later in the code.
    \begin{itemize}
        \item These turn on\textbackslash off listener functions \boldpurp{logData.m} \& \boldpurp{plotData.m}. 
        \item In other words, these are switches you can use to set your code to acquire data without saving it (useful for debugging your circuit) or acquire data without plotting it (can save CPU time, but is risky since you might not notice if something goes wrong with your recording). 
    \end{itemize}  
    \item \boldblue{Filename} \& \boldblue{header} set the file location for writing data, and the text to use for header columns. 
    \begin{itemize}
        \item You need to make sure to create the \boldpurp{.csv} file to write data to in advance of running this code.
    \end{itemize}
    \item  Please use the calibration equation from the previous laboratory experiment. If in doubt, please get in touch with the TAs for the correct calibration. The calibration should be able to convert the potentiometer output voltage to radians. Program this equation into the MATLAB acquisition program. Verify that the calibration is producing the correct output by moving the pendulum to several known angles.
\end{enumerate}

        \end{itemize}
    \end{itemize}
    
    \item Run Data Collection Program
    \begin{itemize}
        \item Run your MATLAB program by clicking the \textit{run} button arrow.
    \end{itemize}
    
    \item Observe Output While Rotating the Pendulum
    \begin{itemize}
        \item While the program is running:
        \begin{itemize}
            \item Rotate the angular potentiometer.
            \item Observe the real-time output on the display figure window.
            \item Keep in mind that the DAQ system takes a few seconds to initialize. You might notice a delay while the simulation starts. 
        \end{itemize}
    \end{itemize}
    
    \item Verify Smooth Voltage Response
    \begin{itemize}
        \item Ensure the voltage changes smoothly over the $360^\circ$ range.
        \item Check to see if voltage for accelerometer at $-90^\circ$ and $90^\circ$ are the same.
        \item If you notice a sudden jump or drop in voltage:
        \begin{itemize}
            \item The potentiometer may be misaligned.
            \item To fix it, ask your TA for help, they can with loosen the grub screw, rotate the potentiometer until the voltage varies smoothly (jump should occur around $+180^\circ$), then re-tighten it.
            \item Re-test to confirm smooth output over the test range.

        \end{itemize}
    \end{itemize}
\end{enumerate}


\subsection{Part 3: Calibration}
When the pendulum is held still, the only acceleration the accelerometer feels is gravity. The voltage read by the accelerometer can be linearly related to acceleration. In this step, you will perform a calibration using four positions & corresponding to the y-axis accelerometer voltage outputs.

\begin{enumerate}
    \item Create the table below in your logbook by rotating the pendulum to the four shown positions and recording the output voltages on the y-axis of the accelerometer. 
    \item Collect the \textbf{single-channel} voltage output from the accelerometer (``AI1"); use 0 or 1 for channel names.
    \begin{itemize}
        \item Duration: 20 seconds
        \item Sample rate: 20 samples per second
        \item Ensure that you have proper slope=1 and intercept=0.
    \end{itemize}
    
    \item Record data at each angle. Each run:
    \begin{enumerate}[label=\alph*.]
        \item Run your program:
        \begin{itemize}
            \item Save a data file using this naming convention: \boldpurp{Ang\_(p/n)XXDeg.csv} where
            \begin{itemize}
                \item \texttt{XX} indicates angle.
                \item \texttt{p/n} indicates sign (positive or negative).
                \item For example: for $-90^\circ$, use \boldpurp{Ang\_n45Deg.mat}.
            \end{itemize}
            \item Update the file name before each run to avoid overwriting data.
        \end{itemize}
        
        \item Record the angular position (in degrees):
        \begin{itemize}
            \item Reference experiment-02 to match angular position.
        \end{itemize}

        \item Compute the Average Voltage:
        \begin{itemize}
            \item Write a MATLAB script to calculate the mean voltage from each data file.
            \item Record the result with at least two decimal places.
        \end{itemize}
    \end{enumerate}
    
    \item Plot the data points from the above table in order to create calibration equations for the y-axis acceleration measurements. Specifically, Plot $ay_g$ vs. $ay_{v,avg}$ to find y-axis acceleration (in $m/s^2$) as a function of sensor voltage.
\end{enumerate}

\subsection{Part 5: Normal acceleration experiment}
\emph {Procedure for collecting data.}
\begin{enumerate}
    \item Move the pendulum to the horizontal position ($90^\circ$).
    \item Start the acquisition program.
    \item Release the pendulum.
    \item Ensure the pendulum comes to a complete stop before your acquisition program ends. 
    \item Make sure the plot shows pendulum angular location and pendulum normal acceleration.
    \item Open your output file and check the data.
    \item Output file should have three columns of data. The first column will have information about time, the second column will have the angular position of the pendulum in degrees, and the third column will have the acceleration value of the accelerometer in $m/s^2$.
\end{enumerate}
