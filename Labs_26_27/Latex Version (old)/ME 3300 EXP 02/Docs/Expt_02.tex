\section{Introduction}

\subsection{Pendulum Model}
An ideal planar pendulum consists of a body suspended from a light rod of length $l$ pinned to a frictionless pivot, which allows it to swing back and forth under the influence of gravity. The dynamics of a pendulum can be simplified significantly by making the following assumptions:
\begin{enumerate}
    \item The mass of the pendulum is concentrated at the swinging end.
    \item The pendulum rod is rigid and massless.
\end{enumerate}
These assumptions result in what is called the model of a "simple pendulum". They will limit the accuracy of the model, but how much error will be introduced? We can use experimental measurements to compare. 

To derive the differential equation of motion for the simple pendulum, refer to Figure \ref{fig:PendulumDiagram}.
The torque about the pivot point P is
\begin{equation}
    {{\bf{\mathord{\buildrel{\lower3pt\hbox{$\scriptscriptstyle\rightharpoonup$}} 
\over \tau } }}_p} = {{\bf{\mathord{\buildrel{\lower3pt\hbox{$\scriptscriptstyle\rightharpoonup$}} 
\over r} }}_{p,m}} \times m{\bf{g}} + {\tau _f} = L{\bf{\hat r}} \times mg\left( {\cos \theta {\bf{\hat r}} - \sin \theta {\bf{\hat \theta }}} \right) - b\dot \theta {\bf{\hat k}} =  - Lmg\sin \theta {\bf{\hat k}} - b\dot \theta {\bf{\hat k}}
\end{equation}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.3\linewidth]{Figures/PendulumDiagram.png}
    \caption{Coordinate System and Torque diagram for a simple pendulum.}
    \label{fig:PendulumDiagram}
\end{figure}

So the z-component of torque about the pivot P  may be written as
\begin{equation}
    {\left( {{\tau _P}} \right)_z} =  - mgL\sin \theta  - b\dot \theta 
\end{equation}

Applying the parallel axis theorem, the moment of inertia of a point mass about the pivot is   so the rotational equation of motion is found as 
\begin{equation}
\begin{array}{l}
{\left( {{\tau _P}} \right)_z} = {I_P}{\alpha _z} = {I_P}\ddot \theta \\
 - mgL\sin \theta  - b\dot \theta  = m{L^2}\ddot \theta .
\end{array}
\end{equation}

One can rearrange to find the standard form as a second-order nonlinear ODE
\begin{equation}
\begin{array}{l}
\ddot \theta  + 2\zeta {\omega _n}\dot \theta  + {\omega _n}^2\sin (\theta ) = 0\\
{\rm{where:}}\\
\begin{array}{*{20}{c}}
{\zeta  = \frac{b}{{2m{L^2}{\omega _n}}},}&{{\omega _n} = \sqrt {\frac{g}{L}} }
\end{array}.
\end{array}
\end{equation}

MATLAB’s ODE 45 can be used to simulate the response of this system, given estimates of the parameters   and   as well as the initial condition (IC) for \boldpurp{Pendulum\_Simulation.m} provides a sample script showing how to implement a simulation of this pendulum model.

\section{Laboratory Instructions}
In this experiment, you will connect a simple angle sensor (angular potentiometer) to the NI USB-6008 data acquisition device and collect digital data using a custom MATLAB data acquisition program. Then, you will collect data to create a calibration equation to convert the output voltage from the angular potentiometer to the pendulum angle. \subsection{Part 1: Prepare your MATLAB Program to Display and Record Data.}
\begin{enumerate}
    \item Launch MATLAB and prepare your code
    \begin{itemize}
        \item A main acquisition script \boldpurp{MATLAB\_Data\_Acquisition.m} is posted on canvas along with two supporting functions \boldpurp{plotdata.m} \& \boldpurp{logData.m}
        \item You need to set the parameters for data acquisition in the main program. They are set near the top of the file.
    \end{itemize} 
    \item Set the value of Fs to set the sample rate (the number of samples to collect per second). 
    \begin{itemize}
        \item The MyDAQ can collect up to 200,000 samples per second, but this would produce a massive data file \& we don't need that much resolution to observe the pendulum swing. 
        \item You will set this value differently for each part of the lab. Note what settings are best for different tasks. 
    \end{itemize}
    \item Set T to control the duration of your acquisition. This tells the DAQ how much time in seconds to collect data.
    \item \boldblue{inputChannels} controls the collected input signals. An example of collecting 2 channels is provided.
    \item The two boolean variables \boldblue{enableLogging} \& \boldblue{enablePlotting} control if statements later in the code.
    \begin{itemize}
        \item These turn on\textbackslash off listener functions \boldpurp{logData.m} \& \boldpurp{plotData.m}. 
        \item In other words, these are switches you can use to set your code to acquire data without saving it (useful for debugging your circuit) or acquire data without plotting it (can save CPU time, but is risky since you might not notice if something goes wrong with your recording). 
    \end{itemize}  
    \item \boldblue{Filename} \& \boldblue{header} set the file location for writing data, and the text to use for header columns. 
    \begin{itemize}
        \item You need to make sure to create the \boldpurp{.csv} file to write data to in advance of running this code.
    \end{itemize}
\end{enumerate}

\subsection{Use your MATLAB program to Test the Circuit and Data Acquisition}
To verify that your circuit is implemented correctly, test it by rotating the potentiometer and confirming the voltage changes as expected at each position. 

\begin{enumerate}
    \item Adjust Duration
    \begin{itemize}
        \item Change Duration to a value that fits your testing needs:
        \begin{itemize}
            \item Use a short duration for quick checks.
            \item Use a longer duration if you want data collection  to run continuously. Note there isn't a built-in way to interrupt collection, so use good judgment! 
        \end{itemize}
    \end{itemize}
    
    \item Run Data Collection Program
    \begin{itemize}
        \item Run your MATLAB program by clicking the \textit{run} button arrow.
    \end{itemize}
    
    \item Observe Output While Rotating the Pendulum
    \begin{itemize}
        \item While the program is running:
        \begin{itemize}
            \item Rotate the angular potentiometer.
            \item Observe the real-time output on the display figure window.
            \item Keep in mind that the DAQ system takes a few seconds to initialize. You might notice a delay while the simulation starts. 
        \end{itemize}
    \end{itemize}
    
    \item Verify Smooth Voltage Response
    \begin{itemize}
        \item Ensure the voltage changes smoothly over the $\pm 90^\circ$ range.
        \item If you notice a sudden jump or drop in voltage:
        \begin{itemize}
            \item The potentiometer may be misaligned.
            \item To fix it, ask your TA for help, they can with loosen the grub screw, rotate the potentiometer until the voltage varies smoothly (jump should occur around $+180^\circ$), then re-tighten it.
            \item Re-test to confirm smooth output over the test range.
        \end{itemize}
    \end{itemize}
\end{enumerate}

\subsection{Part 4: Create Calibration Equation to Output Pendulum Angle}
Next, you will perform a calibration with 10 or more angles. At each angle, you will hold the pendulum steady and collect a short data set. You will repeat this process to sweep the pendulum through the full $\pm 90^\circ$ range incrementally.  

\begin{enumerate}
    \item Determine a reliable method to measure the pendulum angle. Options include:
    \begin{itemize}
        \item The supplied protractor.
        \item A smartphone app such as \textit{Bubble Level}.
    \end{itemize}

    \item Configure your acquisition program by setting:
    \begin{itemize}
        \item Duration: 5 seconds
        \item Sample Rate: 20 samples per second
    \end{itemize}
    \item Create a table in your logbook using the format shown in Table \ref{tab:calibration} to log your calibration measurements.  
    \item Record data at each angle. Each run:
    \begin{enumerate}[label=\alph*.]
        \item Run your program:
        \begin{itemize}
            \item Save a data file using this naming convention: \boldpurp{Ang\_(p/n)XXDeg.csv} where
            \begin{itemize}
                \item \texttt{XX} indicates angle.
                \item \texttt{p/n} indicates sign (positive or negative).
                \item For example: for $-45^\circ$, use \boldpurp{Ang\_n45Deg.mat}.
            \end{itemize}
            \item Update the file name before each run to avoid overwriting data.
        \end{itemize}

        \item Record the angular position (in degrees):
        \begin{itemize}
            \item Use the right-hand rule: point your right thumb out from the potentiometer shaft (see Figure \ref{fig:ZeroPosition}) to determine the positive direction.
            \item Zero degrees is defined as the pendulum pointing straight down.
        \end{itemize}

        \item Compute the Average Voltage:
        \begin{itemize}
            \item Write a MATLAB script to calculate the mean voltage from each data file.
            \item Record the result with at least two decimal places.
        \end{itemize}
    \end{enumerate}
\end{enumerate}

\subsection{Part 7: Overlay a model estimate of pendulum oscillation. }
For this part, you will match a model pendulum simulation to your recorded swing data.

\begin{enumerate}
    \item Open the Provided MATLAB Script
    \begin{itemize}
        \item Refer to \boldpurp{Pendulum\_Simulation.m} for this part.
        \item This script uses \boldpurp{ode45} to simulate a model pendulum’s motion.
    \end{itemize}

    \item Set pendulum length
    \begin{itemize}
        \item Measure your pendulum’s length using a ruler or tape measure.
    \end{itemize}

    \item Set initial parameter estimates
    \begin{itemize}
        \item Without direct measurement, we do not know the values of mass $m$ or the damping coefficient $b$.
        \item Start with a reasonable guess of $m$ and $b$.
    \end{itemize}

    \item Simulate Response with ODE45
    \begin{itemize}
        \item Use \boldpurp{ode45} to simulate the pendulum response using these initial values.
    \end{itemize}

    \item Overlay the model on your pendulum swing plot.
    \begin{itemize}
        \item Use a blue dashed line with a 2~pt linewidth.
    \end{itemize}
    
    \item Refine your model parameters
    \begin{itemize}
        \item Iteratively adjust values for $m$ and $b$ to bring the model into "close" alignment with your experimental data.
        \item You may also need to adjust the model’s initial conditions if your recording did not start with the pendulum at exactly $90^\circ$.
    \end{itemize}
\end{enumerate}
\noindent\textbf{Note:} It is normal for the model and data to show some differences. Aim for approximate alignment, not perfection.
